# Public Endpoints

ANKR, Kava Labs and Chainstack provide public API endpoints for the community to use for consuming and integrating with Kava, e.g. for setting up and syncing a new archive node or getting the latest block info.

## Pruning API

Pruning APIs provide information on the last 1000 blocks, and are useful for getting the latest state and most recent transactions.

### Mainnet Pruning API Endpoints

#### Ankr hosted

Easily add to your wallet with a few clicks [here](https://kava-rpc.com/):

- api.kava-rpc.com - HTTP Rest Endpoint
- rpc.kava-rpc.com - JSON-RPC Endpoint
- evm.kava-rpc.com - EVM JSON-RPC Endpoint
- wevm.kava-rpc.com - EVM Websocket Endpoint

#### Kava Labs hosted:

- api.kava.io - HTTP Rest Endpoint
- rpc.kava.io - JSON-RPC Endpoint
- grpc.kava.io - GRPC Endpoint
- evm.kava.io - EVM JSON-RPC Endpoint
- wevm.kava.io - EVM Websocket Endpoint

#### Chainstack hosted:

- api.kava.chainstacklabs.com - HTTP Rest Endpoint
- rpc.kava.chainstacklabs.com - JSON-RPC Endpoint
- grpc.kava.chainstacklabs.com - GRPC Endpoint
- evm.kava.chainstacklabs.com - EVM JSON-RPC Endpoint
- wevm.kava.chainstacklabs.com - EVM Websocket Endpoint

### Testnet Pruning API Endpoints

#### Kava Labs hosted:

- api.testnet.kava.io - HTTP Rest Endpoint
- rpc.testnet.kava.io - JSON-RPC Endpoint
- grpc.testnet.kava.io - GRPC Endpoint

## Default API Rate Limits

To provide fair use of Kava's public API capacity, request rate limits are enforced for consumers of the following endpoints as below.

#### ANKR hosted:

- Mainnet Pruning API: 7500 requests per 10 minutes

#### Kava Labs hosted:

- Mainnet Archive API: 100 requests per 5 minutes
- Mainnet Pruning API: 1200 requests per 5 minutes
- Testnet Archive API: 500 requests per 5 minutes
